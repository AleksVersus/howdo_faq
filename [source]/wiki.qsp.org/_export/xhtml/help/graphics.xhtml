Графика

Поддерживаемые возможности графического оформления:

  • Картинки для действий. См. ACT.
  • Картинки для элементов меню. См. Меню.
  • Картинки для предметов в инвентаре. См. ADDOBJ.
  • Фоновая картинка основного описания. См. $BACKIMAGE.
  • Показ картинки в отдельном окне с помощью VIEW.
  • Вставка картинок в любую часть текста (например, в основное описание) с помощью HTML-тега IMG.

Подробнее

  • Для действий, предметов и элементов меню изображения задаются при их добавлении.
  • Задать фон можно с помощью переменной $BACKIMAGE, в неё нужно записать путь к картинке. 
    $BACKIMAGE = 'content/back.png'
  • Оператор VIEW позволяет показать картинку в отдельном окошке. 
    VIEW 'content/monster.png'
    • Также мы можем закрыть окно, вызвав VIEW без параметров: 
      VIEW
  • Для того, чтобы вставить картинку в основное или дополнительное описание, можно использовать HTML и тег "IMG". Это самый распространенный способ. 
    ! Включаем режим HTML. Если во всей игре используется HTML, 
! то достаточно включить его на самой первой локации.
USEHTML = 1
! Выводим картинку в основное описание
'<img src="content/room.jpg">'
! Выводим картинку в доп. описание
PL '<img src="content/map.jpg">'
    • Для того, чтобы выровнять такую картинку "по центру", можно воспользоваться тегом "CENTER": 
      '<center><img src="content/room.jpg"></center>'

Внимание! Путь нужно указывать относительно папки с файлом .qsp игры. Например, если файл игры лежит по адресу D:\QSP\game\game.qsp, а картинка - по адресу D:\QSP\game\content\picture.jpg, то путь нужно указывать, как "content/picture.jpg".

Операторы:

  • REFINT - принудительное обновление интерфейса
  • VIEW [$путь] - просмотр картинки из указанного файла.
    • Если вместо [$путь] указана пустая строка ('') или параметр не указан, то это скроет окно с картинкой.
